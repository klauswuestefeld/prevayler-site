Johan Stuyts is a developer interested in: design by contract, naked objects, code generation, object databases (but this might change @:)), user interface design, and lately prevalent systems.

His dreams are:
        * To not have to use SQL databases ever again for storing objects;
        * To build an object database accessible by lots of programming languages, JDBC, and ODBC;
        * To extend the Java programming language with generic types (like Eiffel), design by contract (Eiffel again), block closures (Smalltalk this time), and some other goodies.
