HashCode

This is a rough rendering of a page from the old Prevayler wiki. Please see the new wiki for current documentation.

To be Deterministic, BusinessClasses must pay attention to the proper use of Object.hashCode().

The default Java hashCode() is based on the object's address in the VM heap and cannot be deterministically reproduced. It can only be used as a short-cut to avoid identity comparisons, therefore.

Using the default hashCode() is OK for HashSet.add() and HashSet.remove(), for example. It is NOT OK, though, to use a HashSet.iterator() and process only the first element out of several, for example. If your BusinessClasses do not implement their own hashCode(), the order of the elements in that iterator will vary non-deterministically.

A Set.iterator() can be safely used if all elements are processed regardless of order.

A List.iterator(), as you might expect, is always safe to use.
