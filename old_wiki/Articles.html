A better representation would be each system have an object space with exactly the same contents as the other. -- KlausWuestefeld

The code in the "Introduction to Object Prevalence" article has a little glitch, pointed out by several readers. In listing 2, an object called UserLogonSystem is created that extends AbstractPrevalentSystem. Then in listing 3 another object "system" is cast from PrevalentSystem to UserManager (not the UserLogonSystem we created in listing 2) and a function called getUsers, much like the one on UserLogonSystem, is called. I've noticed that error, but it was just too late to fix it. Sorry guys! @:( -- CarlosVillela
