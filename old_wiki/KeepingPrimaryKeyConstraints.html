KeepingPrimaryKeyConstraints

This is a rough rendering of a page from the old Prevayler wiki. Please see the new wiki for current documentation.

How does one enforce primary key constraints when using Prevayler?  If a new object is created every time with the same id given to it, now a query for a particular id returns more than one object.  That is very bad!  Forcing coders to just make sure they don't use an id that is already in existence is not a good idea. It is easy to make a mistake. That is why databases allow enforcement of unique fields. That way, even the laziest of developers won't make a mistake. Is there functionality in Prevayler that allows for this enforcement? -- JacobKjome, by mail

See: AutoIncrement.
