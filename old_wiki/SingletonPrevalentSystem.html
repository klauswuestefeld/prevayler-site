SingletonPrevalentSystem

This is a rough rendering of a page from the old Prevayler wiki. Please see the new wiki for current documentation.

Using the singleton pattern on an AbstractPrevalentSystem subclass gives a great deal of flexibility when you need to have access to the prevalent system in the client classes. Let's exemplify:

Suppose you're creating a class, named Utils, to help you do that little queries needed inside your commands. You can do this by passing a PrevalentSystem instance to the method, which couples it's interface to Prevayler:

public static User findUser(String login, PrevalentSystem system) {
  // ...
}

Or you can have a singleton prevalent system, and do it like this:

public static User findUser(String login) {
  MySystem system = MySystem.getInstance();
  // ...
}
